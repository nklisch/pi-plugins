# Harden hook adaptation with golden and order evidence

## Checkpoint

Prove the complete process-free path from verified runtime snapshots and unchanged real hook declarations through fake Pi boundaries to exact ordered event plans, while locking down compatibility coverage, package exports, dependency direction, and migration-free rollback.

## Design element

- Add a complete runtime golden fixture combining the real Agile Workflow hooks with a verified skill/hook/MCP projection and user/project catalog snapshots.
- Exercise every supported ordinary event, Pi session/compaction reason, matcher/condition form, transcript mode, tool success/failure/interrupt shape, and Stop continuation state.
- Run event-order scenarios for startup, prompt, parallel tools, manual/threshold/overflow compaction and retry, settled Stop, reload, new, resume, fork, and quit.
- Assert canary handlers are never spawned and Pi-only evidence never escapes the namespace.
- Keep planner/root/signal/regex/Pi adapter/fake types out of `src/index.ts` and compiled exports. Update rolling foundation assertions only when landed behavior makes current prose false or misleading.

## Acceptance evidence

- The unchanged `test/fixtures/plugins/hooks/agile-workflow-hooks.json` selects exact session, compact, and `Write|Edit|apply_patch` plans with full stdin goldens.
- Exhaustive contract tables fail when an event, alias, condition operator, Pi boundary, source, or trigger lacks both positive and negative evidence.
- Event-order tests prove completed compaction dispatches `PostCompact` before compact `SessionStart`, Stop follows all automatic work, and parallel completion never changes selected-handler order.
- Negative tests cover invalid regex, unsupported condition/event fields, non-JSON evidence, permission approximation, stale trust/project context, selector corruption, and projection mismatch without partial plans.
- Diagnostics and foreign stdin omit canary secrets, native causes, plugin/data roots, unavailable transcript/permission/result fields, and raw Pi-only keys.
- Source/compiled API and dependency-boundary tests prove no execution-capable or raw-root API escaped.
- Full `npm test` passes typecheck, boundaries, Vitest, build, and exact compiled import; baseline/addition counts are recorded in the feature body.
- Rollback requires no state, projection, trust, transition, content, data, or recovery migration; removing this derived capability leaves command hooks unavailable before activation.

## Ordering constraint

Depends on the Pi lifecycle bridge and closes the feature implementation checkpoint graph. It is integration evidence, not a separate command-execution phase.
